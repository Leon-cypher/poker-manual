## 🎯 核心概念：下注判定的三大原則

::: tip
💡

**記住這三句話：**

1. **口頭說的算數** — 說了就要照做，籌碼只是執行
2. **過 50百分比 算加注意圖** — 超過基準點就要補到最小加注
3. **一次動作原則** — 第一次推出去的才算，不能反悔
:::

---

## 📋 判定流程圖

```
玩家推出籌碼
     ↓
有口頭宣告嗎？
     ↓
   有 → 以口頭為準（籌碼只是執行）
     ↓
   沒有 → 看推出幾顆籌碼？
     ↓
【單顆籌碼】→ 判定為跟注（無論面額多大）
     ↓
【複數籌碼】→ 移除一顆最小面額
     ↓
   剩餘金額足夠跟注嗎？
     ↓
   不夠 → 判定為跟注
     ↓
   足夠 → 計算「目前下注額 + 級距 × 50百分比」
     ↓
   推出金額 ≥ 基準點 → 判定為加注，補到最小加注
   推出金額 < 基準點 → 判定為跟注
```

## 🔥 情境案例：桌上會遇到的狀況

### 案例 1：單顆大額籌碼（最常見）

**情境：**

- 玩家 A 下注 1500
- 玩家 B **沒說話**，推出一顆 5000

**判定：**

✅ **跟注 1500**（單顆籌碼 = 跟注，無論面額多大）

**找零：**

- 先從底池拿 3500 給玩家 B
- 再將 5000 收入底池

---

### 案例 2：複數籌碼 — 剛好跟注

**情境：**

- 玩家 A 下注 2400
- 玩家 B 推出 2 個 1000 + 1 個 500

**判定流程：**

1. 移除最小面額（500）→ 剩 2000
2. 2000 < 2400（不足跟注）
3. ✅ **判定為跟注 2400**

**找零：**

- 先從底池拿 100 給玩家 B
- 再將 2500 收入底池

---

### 案例 3：複數籌碼 — 加注意圖（重點）

**情境：**

- 玩家 A 下注 1500
- 玩家 B 推出 1 個 5000 + 1 個 500

**判定流程：**

1. 移除最小面額（500）→ 剩 5000
2. 5000 > 1500（足夠跟注）
3. 計算基準點：1500 + 1500 × 50百分比 = **2250**
4. 5000 > 2250 → **有加注意圖**
5. 最小加注 = 1500 + 1500 = **3000**
6. 玩家實際推出 5500 > 3000
7. ✅ **判定為加注 5500**

**找零：**

- 不需要找零，直接收 5500 入底池

---

### 案例 4：口頭宣告 vs 籌碼不符

**情境：**

- 玩家 A 下注 2000
- 玩家 B 說「Raise 6000」，但只推出 5000

**判定：**

✅ **以口頭為準：加注到 6000**

- 玩家 B 需補 1000

**反過來：**

- 玩家 B 說「Call」，但推出 3 個 5000
- ✅ **以口頭為準：跟注 2000**
- 找零 13000 給玩家 B

---

### 案例 5：短碼 All-in 關閉級距（進階）

**情境：**

- 玩家 A 下注 5000
- 玩家 B 跟注 5000
- 玩家 C（短碼）All-in 9000

**判定：**

- 級距 = 5000（上一個合法級距）
- 增量 = 9000 - 5000 = 4000
- 4000 < 5000（增量 < 級距）
- ✅ **級距關閉，玩家 A 和 B 不能再加注**

**如果 C All-in 11000：**

- 增量 = 11000 - 5000 = 6000
- 6000 > 5000（增量 ≥ 級距）
- ✅ **級距打開，A 可加注，最小加注 = 11000 + 5000 = 16000**

## ❌ 常見錯誤 & 地雷區

::: tip
⚠️

**新手最容易搞錯的 5 件事：**

❌ **錯誤 1：看到多顆籌碼就直接算總額**

✅ 正確：先移除一顆最小面額再判斷

❌ **錯誤 2：單顆 5000 跟注 1500，以為是加注**

✅ 正確：單顆籌碼 = 跟注（無論多大）

❌ **錯誤 3：超過跟注額就算加注**

✅ 正確：要超過「目前下注額 + 級距 × 50百分比」才算

❌ **錯誤 4：玩家說完 Raise 又說 Call**

✅ 正確：以第一次宣告為準（一次動作原則）

❌ **錯誤 5：All-in 9000 就能讓後面的人加注**

✅ 正確：增量 < 級距 = 關閉，增量 ≥ 級距 = 打開

:::

## 💡 老手提醒：實戰經驗

::: tip
🧠

**提醒 1：找零先給，再收籌碼**

新手常犯的錯誤是先把籌碼收走，玩家才發現找零不對。正確流程是：

1. 先從底池拿找零給玩家
2. 讓玩家確認金額
3. 再將籌碼收入底池

**提醒 2：看不清楚就請玩家重新擺放**

玩家推出一堆籌碼但擺放混亂、看不清數量或面額時：

「請您將籌碼重新擺放，讓我確認一下。」

「麻煩稍等，我確認一下您推出的籌碼。」

❌ **錯誤做法：** 問玩家「您是要跟注還是加注？」（這等於讓玩家事後選擇）

✅ **正確做法：** 確認籌碼數量和面額後，根據規則直接判定

**提醒 3：口頭宣告要聽清楚**

「Call」和「All」發音很像，尤其在吵雜環境。

沒聽清楚就問：「請再說一次？」

**提醒 4：級距計算不用當場算給玩家聽**

玩家問「為什麼我要補到 3000？」

簡單回答：「因為您推出的金額超過跟注的 1.5 倍，依規則需補到最小加注。」

不用現場教數學，保持節奏。

:::

### 題目 1：基礎判定

**情境：**

盲注 200/400，玩家 A 下注 1000，玩家 B 推出 2 個 1000

**問：這是跟注還是加注？金額多少？**

<details>

<summary>💡 點擊查看答案</summary>



**答案：跟注 1000**

**判定流程：**

1. 複數籌碼 → 移除最小面額（1000）
2. 剩 1000 = 跟注額
3. 判定為跟注
4. 找零 1000 給玩家 B

</details>

---

### 題目 2：50百分比 基準點

**情境：**

玩家 A 下注 2000，玩家 B 推出 1 個 5000 + 1 個 1000

**問：這是跟注還是加注？金額多少？**

<details>

<summary>💡 點擊查看答案</summary>



**答案：加注 4000**

**判定流程：**

1. 複數籌碼 → 移除最小面額（1000）
2. 剩 5000 > 2000（足夠跟注）
3. 基準點 = 2000 + 2000 × 50百分比 = **3000**
4. 6000 > 3000 → 有加注意圖
5. 最小加注 = 2000 + 2000 = **4000**
6. 判定為加注 4000
7. 找零 2000 給玩家 B

</details>

---

### 題目 3：口頭 vs 籌碼

**情境：**

玩家 A 下注 3000，玩家 B 說「Raise 8000」，但推出 1 個 10000

**問：最終下注額是多少？**

<details>

<summary>💡 點擊查看答案</summary>



**答案：加注 8000**

**原因：**

口頭宣告優先於籌碼

**處理：**

1. 玩家 B 加注到 8000
2. 找零 2000 給玩家 B
3. 收 10000 入底池

</details>

---

### 題目 4：短碼 All-in（進階）

**情境：**

玩家 A 下注 10000，玩家 B All-in 16000，玩家 C 想加注

**問：玩家 C 的最小加注是多少？**

<details>

<summary>💡 點擊查看答案</summary>



**答案：22000**

**判定流程：**

1. 級距 = 10000（上一個合法級距）
2. 增量 = 16000 - 10000 = 6000
3. 6000 < 10000（增量 < 級距）
4. **級距關閉，玩家 C 不能加注**

**等等，題目問的是「想加注」：**

- 如果增量 ≥ 10000，才能打開級距
- 假設 B All-in 20000 以上，C 才能加注
- 最小加注 = 20000 + 10000 = 30000

**實際上 B 只 All-in 16000：**

- ✅ **C 只能跟注或棄牌，不能加注**

</details>

## 🔍 快速查詢：關鍵詞索引

- **單顆籌碼** → 永遠是跟注
- **複數籌碼** → 移除最小面額再判斷
- **50百分比 基準點** → 目前下注額 + 級距 × 50百分比
- **最小加注** → 目前下注額 + 級距
- **口頭優先** → 說了算，籌碼只是執行
- **一次動作** → 第一次推出才算，不能反悔
- **All-in 關閉級距** → 增量 < 級距
- **All-in 打開級距** → 增量 ≥ 級距
